## GitHub Setup Instructions

### Step 1: Create GitHub Repository
```bash
# If you haven't already, initialize git in your project
git init

# Add all files
git add .

# Make initial commit
git commit -m "Initial commit: Spotify usability improvements"

# Create repository on GitHub (github.com)
# Then connect your local repo to GitHub:
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git
git branch -M main
git push -u origin main
```

### Step 2: Verify Repository Contents
Make sure your GitHub repository includes:
- [ ] All source code files
- [ ] `README.md` (project documentation)
- [ ] `package.json` (dependencies)
- [ ] `.gitignore` file
- [ ] Clear instructions on how to run the project

### Step 3: Test the Repository
1. Clone your repository to a new location
2. Follow your own README instructions
3. Verify the project runs correctly
4. Fix any issues and push updates

### Step 4: Make Repository Public
- Go to repository Settings
- Scroll to "Danger Zone"
- Click "Change visibility"
- Select "Public"
- This ensures your instructor can access it

## Presentation Slides Preparation

### Slide Content Requirements
Your slides MUST include:
- [ ] Summary of Project 1 usability issues
- [ ] Solutions for each usability issue (minimum 2)
- [ ] Explanation of design principles applied
- [ ] Demo (live or pre-recorded)
- [ ] Technical challenges encountered

### Slide Format
- **Accepted**: .ppt, .pptx, or .pdf
- **Recommended**: Export as PDF for compatibility
- **File name**: `Team#_Project2_Presentation.pdf`

### Embedding Demo in Slides
If pre-recording demo:
- Use screen recording software (Loom, OBS, QuickTime, etc.)
- Keep demo under 3 minutes
- Embed video in PowerPoint, or
- Upload to YouTube/Vimeo and include link in slides
- Have video file as backup

## Canvas Submission Process

### Before Submission
1. **Test your GitHub link**
   - Open in incognito/private browser
   - Verify it's accessible
   - Check all files are visible

2. **Test your slides**
   - Open on different computer if possible
   - Verify all images/videos display
   - Check file size (Canvas may have limits)

3. **Prepare submission text**
   ```
   Team Members: [Name 1, Name 2, Name 3, Name 4, Name 5]
   
   GitHub Repository: [https://github.com/...]
   
   Project Description:
   This project addresses 5 usability issues identified in Project 1:
   1. Adding songs to playlists - Improved confirmation feedback
   2. Download state indicators - Clear streaming vs downloaded status
   3. "Newest" album visibility - Prominent badge design
   4. Artist discography organization - Search and category filters
   5. Search result ordering - Songs appear first
   
   Technology Stack: React, TypeScript, Tailwind CSS
   
   To run the project:
   1. Clone the repository
   2. Run: npm install
   3. Run: npm run dev
   4. Open browser to localhost
   ```
